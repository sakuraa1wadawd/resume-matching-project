# 项目报告：基于大数据+AI的简历-岗位人才匹配系统

## 1. 项目背景

企业招聘中会产生大量简历和岗位文本，人工筛选效率低且标准不统一。本项目使用 Python 文本处理、机器学习相似度算法、规则评分、Streamlit 可视化、HDFS 数据管理和 PySpark 清洗流程，实现一个可运行、可解释的人岗匹配系统。

## 2. 数据设计

简历数据包含 `resume_id`、姓名、学历、专业、技能、经验年限、期望城市、期望薪资、证书、项目经历、自我描述等字段。岗位数据包含 `job_id`、岗位名称、公司、最低学历、技能要求、经验要求、城市、薪资、偏好证书、岗位描述等字段。

## 3. 文本预处理

系统先将简历的专业、技能、证书、项目经历、自我描述拼接成简历文本，将岗位名称、岗位技能、证书偏好、岗位描述拼接成岗位文本。随后使用 jieba 分词、停用词过滤和 `skill_alias.json` 技能词标准化，将不同写法统一为标准技能词，例如将 `Python语言` 统一为 `python`，将 `SparkSQL` 统一为 `spark`。

## 4. 相似度算法

TF-IDF 用于衡量词面重合和关键词重要性，适合发现简历和岗位中共同出现的核心词。Word2Vec 用于生成词向量，再通过平均词向量表示整段文本，适合衡量语义相似度。两类向量均使用余弦相似度计算分数，并映射到 0-100。

## 5. 多维评分

系统实现技能分、TF-IDF 分、Word2Vec 分、学历分、经验分、城市分、证书分、薪资分。综合分公式为：

```text
技能分 × 0.28 + TF-IDF分 × 0.20 + Word2Vec分 × 0.15 + 学历分 × 0.12
+ 经验分 × 0.10 + 城市分 × 0.05 + 证书分 × 0.05 + 薪资分 × 0.05
```

## 6. 推荐结果解释

推荐理由由规则自动生成，包含共同技能、缺少技能、学历和经验是否满足、城市是否匹配、证书情况以及文本语义相似度等级。例如：“共同技能包括 python, sql, 数据分析；缺少 excel，建议补充相关练习或项目经验；学历满足岗位要求；经验满足岗位要求；期望城市与岗位城市一致；文本语义相似度较高。”

## 7. 系统实现

本地版由 `src/matcher.py` 调度，输出 `output/explainable_match_result.csv`、`output/top_matches.csv` 和 `output/job_candidate_matches.csv`。展示层使用 `app.py`，支持学生视角、岗位视角、最低分筛选、Top-N 调整和 CSV 下载。

## 8. HDFS 与 PySpark

HDFS 用于保存原始数据和清洗结果，目录为 `/resume_matching/raw_data`、`/resume_matching/cleaned_data` 和 `/resume_matching/results`。`src/spark_job.py` 使用 PySpark 从 HDFS 读取 CSV，完成去重、空值填充、文本拼接和清洗，并将结果写回 HDFS。

## 9. Word2Vec 迁移说明

本地版使用 gensim Word2Vec 训练词向量，并对每份简历和岗位的分词结果取平均词向量，再计算余弦相似度。迁移到 Spark MLlib 时，可以将 `tokens` 列作为输入，使用 `pyspark.ml.feature.Word2Vec` 训练分布式词向量，得到 `resume_vector` 和 `job_vector` 后继续使用余弦相似度计算语义分。由于当前样例数据规模较小，本项目先保留本地 Word2Vec 作为算法演示，PySpark 负责 HDFS 数据读取、清洗和写回。

## 10. 项目不足与改进

当前样例数据规模较小，Word2Vec 训练语料有限，语义效果会受影响。后续可以扩充真实简历和岗位数据，使用 Spark MLlib 实现分布式 TF-IDF 与 Word2Vec，并加入更细粒度的岗位类别、项目经验质量评分和证书权重。
